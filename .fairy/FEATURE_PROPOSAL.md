# 妖精尾巴「智能协作白板」方案

## 三个模块，按优先级

### P0 — 共享工作区状态 《白板》
每个项目维护一个状态文件，所有 Agent 读写，彻底解决"信息靠传话"的问题。

**文件结构：**
```
workspace/{project}/.fairy/
├── state.json       ← 当前工作状态
├── changelog.md     ← 每次变更记录
└── review.json      ← 代码审查结果
```

**state.json 格式：**
```json
{
  "project": "watermark-remover",
  "phase": "开发",
  "currentAgent": "纳兹",
  "nextAgent": "格雷",
  "chainBroken": false,
  "lastAction": {
    "agent": "阿悄",
    "action": "分配开发任务",
    "time": "2026-05-06T16:00:00+08:00",
    "files": ["index.html"]
  }
}
```

**效果：**
- 纳兹改完 → 更新白板 "改了哪些文件、改了什么"
- 格雷读白板就知道要测什么
- 马卡罗夫巡警读白板就知道任务链状态
- 我不用每次都翻聊天记录

---

### P1 — 代码知识索引 《记忆》
纳兹每次写代码要读整个文件，效率低。建一个代码索引，让他先看索引再改具体函数。

**实现：** 每次代码修改后自动更新索引
```
workspace/{project}/.fairy/code-index.json
```
包含：
- 文件结构树
- 每个文件的关键函数/类/组件
- 数据流（哪个文件调了哪个）
- 最后修改时间

**效果：** 纳兹开发效率翻倍，不用从头读大文件

---

### P2 — 自动代码审查 《温蒂》
纳兹改完代码后，温蒂自动审查：
1. 安全性：SQL注入、XSS、敏感信息泄露
2. 代码质量：重复代码、过长函数
3. 常见 bug：null 引用、未处理错误
4. 结果写入 review.json

**效果：** 格雷测试前先修一轮问题，减少迭代次数

---

## 实现路径

**第一天：P0 白板**
- 建 `.fairy/` 目录
- 写 state.json 读写脚本
- 各 Agent prompt 加上"先读白板"
- 马卡罗夫巡警加上白板检查

**第二天：P1 索引**
- 写代码索引生成脚本（找关键函数、类、组件）
- 每次纳兹改完后更新
- 纳兹 prompt 加上"先读索引"

**第三天：P2 审查**
- 温蒂上线，配置审查规则
- 纳兹完成 → 自动触发温蒂审查
- 审查结果写入 review.json
