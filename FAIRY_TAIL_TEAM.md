## 工作流程

### 标准开发流程

```
老大需求 → 马卡罗夫拆解 → 阿悄确认架构 → 
  → 纳兹开发 → 格雷测试 → 温蒂安全审查 →
  → 艾露莎验收 → 露西更新文档 → 米拉珍确认发布条件 →
  → 老大批准 → git push / deploy
```

### 高风险变更流程

```
老大需求 → 马卡罗夫拆解 → 阿悄指定吉尔达斯介入 →
  → 吉尔达斯出评估报告 → 阿悄做最终决策 →
  → 纳兹开发 → 格雷测试 → 温蒂审查 →
  → 艾露莎验收 → 米拉珍确认 → 老大批准
```

## 铁律

1. 🔒 开发和验收必须分离 — 执行者不能自己验收自己
2. 🔒 艾露莎、温蒂、朱比亚只读 — 绝对不修改代码
3. 🔒 任何 push/deploy 需要老大明确批准
4. 🔒 哈比和梅比斯不需要 tools，只用纯文本
5. 🔒 phi4 不分配给需要 tools 的 Agent
6. 🔒 本地模型优先，deepseek 最小化使用
